# Business Wiki Schema

## 이 위키의 목적

코스메틱 산업 분석과 회사 업무 지식을 엔티티·개념 단위로 축적하는 지식 베이스.
소스 1개 = 요약 1개가 아니라, 소스에서 추출한 지식이 해당 엔티티·개념 페이지로 분산 통합된다.

## 폴더 구조

```
raw/          — 원본 소스. 절대 수정 금지. 불변.
  perplexity/ — Perplexity 리서치 결과
  clipped/    — 웹 클립
  docs/       — PDF 등 문서
  assets/     — 이미지 등
wiki/
  entities/   — 인물, 기업, 장소, 제품 (고유명사)
  concepts/   — 전략, 트렌드, 이론 (추상 개념)
  events/     — 특정 시점의 사건, 의사결정
  comparisons/ — 비교 분석 (쿼리 결과 저장)
  guides/     — 스타일 가이드, 운영 지침
  content/    — 콘텐츠 초안·발행 기록
  analysis/   — 복합 분석 (이관된 기존 파일)
  career/     — 커리어 관련 노트
index.md      — 전체 카탈로그
log.md        — 작업 로그 (시간순, append-only)
```

## 페이지 템플릿

### 엔티티

```
---
title: 
type: entity
sources: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# 제목

## 요약
한 문단 핵심 요약.

## 핵심 데이터
테이블 형식. 설립, 매출, 가격대 등.

## 상세
소스에서 추출한 팩트. 출처는 [^소스명] 각주.

## LAKA 시사점
LAKA에게 주는 교훈 또는 비교 포인트.

## 연결
- [[관련_엔티티]] — 관계 설명
```


## 인제스트 워크플로

소스 1개를 처리할 때 아래 6단계를 순서대로 수행한다.

### 1단계 — 소스 읽기 & 지식 목록 추출
소스에서 다음을 뽑는다:
- 엔티티 (인물, 기업, 장소, 제품)
- 개념 (전략, 트렌드, 이론)
- 사건 (특정 시점 이벤트)
- 관계 ("A는 B의 파트너다")
- 수치/데이터 (매출, 시장 규모 등)

추출 목록을 먼저 보여주고 확인받는다.

### 2단계 — 기존 페이지 매핑
index.md를 읽고 각 항목이 이미 위키에 있는지 확인.
- 있으면 → 해당 페이지 업데이트
- 없으면 → 새 페이지 생성

### 3단계 — 페이지 업데이트 / 생성
업데이트 시:
- 기존 내용과 충돌하면 `> ⚠️ 출처 간 충돌:` 블록으로 양쪽 모두 남김
- 새 정보는 해당 섹션 뒤에 추가, `[^소스명]` 각주 표기
- frontmatter sources 리스트에 소스명 추가
- updated 날짜 갱신

절대 하지 않는 것:
- src_소스명.md 같은 소스별 요약 페이지 생성 금지
- 소스의 모든 내용은 엔티티/개념 페이지로 분산 통합

### 4단계 — 위키링크 [[]] 삽입
다른 위키 페이지로 연결 가능한 고유명사에 [[링크]] 삽입.
index.md에 있는 페이지만 링크한다.

### 5단계 — index.md 업데이트
새 페이지는 해당 섹션에 추가. 총 페이지 수 갱신. 날짜 갱신.

### 6단계 — log.md 기록
```
## [YYYY-MM-DD] ingest | 소스명
- 생성/업데이트한 페이지 목록
- 발견된 충돌 사항
```

## 검색 규칙

1. 위키 검색 시 index.md를 먼저 읽는다.
2. 관련 페이지만 열어서 컨텍스트로 사용한다.
3. wiki/ 전체를 한 번에 읽지 않는다.

## 금지 사항

1. src_*.md 소스별 요약 페이지 생성 금지
2. raw/ 파일 수정 금지
3. 존재하지 않는 페이지로 [[링크]] 걸기 금지
4. 인제스트 없이 index.md 수동 편집 금지
