# 5R 프레임워크 — AI 이니셔티브 운영 시스템

> Ayelet Israeli & Eva Ascarza(HBS Digital Data Design Institute, Customer Intelligence Lab)가 익명 라틴아메리카 대형 복합기업의 내부 시스템을 분석해 정리. 원래 데이터 기반 문화 정착용이었으나 AI 이니셔티브 확장의 운영 시스템으로 확장 적용됨.

원전: [[src_HBR_5R_AI_framework_2025]]

## 핵심 명제

> "AI 이니셔티브가 실패하는 건 모델이 부실해서가 아니다. 조직이 이를 지속할 구조를 갖추지 못했기 때문이다."

AI 파일럿이 정체되는 이유 = 데이터 문화 정착이 실패하는 이유 = **불명확한 소유권 + 파편화된 실행 + 약한 책임 구조 + 낮은 수용도**. 5R은 이 4가지를 정면 공략.

## 다섯 구성 요소

### 1. Roles (역할)
프로젝트 생애주기 전반에 걸쳐 누가 무엇을 담당하는지 사전 정의.

| 역할 | 책무 |
|---|---|
| 비즈니스 스폰서 | 가치 실현 책임 (출시 ≠ 종료) |
| 제품 책임자 | 일상 진행·우선순위 |
| 데이터 사이언티스트 | 모델 개발·검증 |
| **번역자 (Translators)** | 비즈니스 ↔ 기술 인터페이스 |
| 리스크·컴플라이언스 | 거버넌스 통제 |
| CX 책임자 | 수용도·고객 경험 |

→ 인계 실패·책임 공백·기술vs비즈니스 갈등 방지.

### 2. Responsibilities (책임)
**최초 출시 이후**까지 각 역할의 성공 기준 정의.

- 수용도, KPI 소유권, **모니터링, 재학습** 포함
- 스폰서는 **승인부터 가치 측정까지** 책임
- "Plug-and-play 도구가 아니다" — 지속적 학습 모델은 누군가의 명시적 업무
- 표준 실패 모드 차단: 파일럿 이후 가치 증발, 담당자 없는 모델 드리프트

### 3. Rituals (루틴)
일관된 상호작용 리듬.

- 프로젝트 킥오프
- **주간 운영 리뷰**
- **격주 임원 위원회** (권한 있는 인사 참여로 부서 간 장애 제거)
- **출시 후 모니터링 회의** ← 책임 있는 AI의 핵심 통제 포인트

→ 일관된 업데이트 습관, 실시간 반복·에스컬레이션.

### 4. Resources (자원)
재사용 가능 템플릿·프레임워크·가속화 도구 **의무화**.

- 공유 생성형 AI 아키텍처: 대규모 배포 복잡성 해소 + 환각/정보 손실 같은 리스크 완화
- 임기응변 대비 **납기 50~60% 단축**
- 거버넌스·책임 있는 AI 관행이 일관 적용되는 표준 인프라

### 5. Results (성과)
프로젝트 시작 전 **수용도 + 비즈니스 임팩트** 지표 동시 정의 의무.

- 모델 정확도 같은 기술 지표·vanity 지표 **만으로는 부족**
- 예: GenAI 처리 상호작용 비율 / churn 감소 / EBITDA / 마케팅 전환율
- 데모만 만들고 KPI 없이 끝나는 패턴 차단

## 구조적 특성

```
표준화된 중추 (Backbone)      유연한 적용 (Application)
━━━━━━━━━━━━━━━━━━━━━━━━━     ━━━━━━━━━━━━━━━━━━━━━━━━━
Roles                         Resources
Responsibilities              Results
Rituals
```

담당자가 교체되어도 중추는 유지. → 확장 가능성의 근원.

## 검증 사례

### Case 1 — 금융 서비스 가격 책정 자동화
- 스폰서는 AI 모델 개선을 성장 목표와 연결
- KPI: 운영 비용 + 고객 리스크 세그먼트별 구매율
- 주간 운영 리뷰 + 격주 임원 위원회로 부서 간 장애 제거
- 표준화 자원 활용 → **임기응변 대비 절반 시간**
- 결과: **동일 매출 유지 + 리스크 비용 8%↓** → 수익성 향상
- → 전사 신규 프로젝트 5R 표준 채택

### Case 2 — 중견 기업 전사 AI 전환
- CEO 직접 후원
- 운영위원회 + CEO 주도 전략위원회 (핵심 루틴) 신설
- 18개월 결과:
  - Churn 감소, 마케팅 캠페인 전환율 향상, 1인당 비즈니스 볼륨 증가
  - 분기당 **100+** 비즈니스 요청 처리
  - 고객 서비스 GenAI 처리율 **3% → 60% (6개월)**
  - 모델 정확도 **92% → 97%**
  - 다른 자회사로 학습 결과 복제

## 책임 있는 AI 통합

5R은 별도의 책임 AI 레이어가 아니라, AI 거버넌스가 **운영 루틴에 박힌** 구조.

- **Rituals** → 출시 후 지속 감독·모니터링: 모델 드리프트(예: 성별에 따른 대출 승인률 차등) 조기 포착
- **Resources** → 거버넌스·컴플라이언스 검토를 프로젝트 라이프사이클 필수 단계로 내재
- **Results** → 재무 성과와 공정성·컴플라이언스·고객 신뢰 균형 설계
- 산출물: 지속 모니터링 보고서, 감사 가능한 결과물

## 어디에 쓰는가

- AI 파일럿 → 운영 → 전사 성과 전환의 운영 OS
- 데이터 기반 문화의 행동·인센티브·기대 정렬
- "표준화된 중추 + 유연한 적용"이 필요한 모든 변환 이니셔티브에 응용 가능 (저자는 AI 영역에서 검증)
