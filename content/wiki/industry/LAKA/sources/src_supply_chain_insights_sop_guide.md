## 소스 개요

Supply Chain Insights LLC가 2019년 1~3월 기술 브리핑 + 같은 해 100명+ 정량 서베이 + 50명+ 비즈니스 사용자 인터뷰를 종합한 실무 가이드. **Implement 핸드북(컨설팅 베스트프랙티스 관점)과 달리 정량 데이터·성숙도 모델·벤더 카탈로그 중심**이라는 게 차별점.

**이 문서의 핵심 주장 (3가지):**
> 1. *"S&OP는 60% 변화관리, 30% 프로세스 정의, 10% 기술 구현이다. 그러나 대부분 기업은 기술에 80%를 쓴다."* (p5)
> 2. *"2016년 65%가 자사 S&OP를 effective로 평가했지만 2019년엔 35%로 하락. 이유는 비즈니스 변화가 아니라 프로세스 진화의 함정에 빠졌기 때문."* (p11)
> 3. *"57%의 기업이 S와 OP가 균형을 잃은 상태. 41%만 균형 상태."* (p15)

## 섹션별 핵심 내용

### Executive Overview (p5)
- ROI: 평균 7개월에 본전 회수
- S&OP 성숙도 ↔ Price-to-Tangible Book Value (PTBV) 상관관계 존재
- 평균 제조사가 S&OP에 **7년+ 투자했지만 38%만 만족**
- 변화관리 60% / 프로세스 30% / 기술 10% 비중이 정답, 외부 컨설턴트로 내부 리더십 갭 메울 수 없음

### What Is S&OP? (p6-8)
- **4단계 시간 호라이즌**:
  | 호라이즌 | 범위 | 초점 |
  |----------|------|------|
  | Strategic / Network Design | 수개월~수년 | 자산·운송 전략, S&OP의 인풋 |
  | **Tactical (= S&OP)** | 주~월 | 주문/재고 매칭~집합 자재계약 |
  | Operational | 일~주 | DRP·MEIO·생산계획·MRP |
  | Executional | 시간~일 | OMS·ATP·운송·생산실행 |
- **Above the Line vs Below the Line**: "Line" = S&OP 임원 검토 회의(MT Review). Above = S&OP 자체, Below = 자재·자산 중심 실행 계획. 지난 10년간 Above는 발전했으나 Below가 더 messy해짐
- **3대 흔한 실수**: ①단기 슬러시 기간 집착(Reactive) ②예산을 인풋으로만 다루고 통합 못함(Role of Budget) ③개선 사이클 부재(Lack of Discipline)
- **"통합(Integration)은 함정"**: Sales/Finance/SCM 데이터 모델은 본질적으로 정렬 안 됨. 답은 **Synchronization + Harmonization** (시각화로 가정 영향 모델링)

### Definitions Matter — IBP vs S&OP (p9-13)
- IBP는 2005년 Oliver Wight 도입. 일관 정의 없음 — 일부는 "Sales+Finance+SCM 연결", 일부는 "S&OP의 성숙 형태"
- **본 보고서 입장: IBP = 5단계 maturity model의 Stage 3** (S&OP의 더 성숙한 형태)
- 3개 기능 데이터 정의 차이:
  | 차원 | Sales | Finance | Supply Chain |
  |------|-------|---------|--------------|
  | Data Definition | 파이프라인 기회의 계약 가치 | 통화 단위 high-level 집계 | 출하 최세부 단위 볼륨 |
  | Planning Level | 고객/위치 (Ship-to) | 카테고리/브랜드 | 출하지 단위 SKU |
  | Focus | 고객 기회·파이프라인 | 비용·수익성 | 가용 계획·서비스 |
- **Pothole vs Pitfall** (p13):
  | POTHOLES (점진적 열화) | PITFALLS (구조적 함정) |
  |-------------------------|--------------------------|
  | 계획 엔진 효과성 저하 | 전략 정렬 실패 |
  | 교육·스킬 희석 | 예산의 역할 |
  | 마스터 데이터 관리 | 조직 거버넌스 |
  | 프로세스 정의의 명확성 | 프로세스 규율·역할 명확성 |
  | 리더십 변동 | 측정 시스템 |
- **6개 핵심 정의** (p13):
  - **Bullwhip**: 수요 신호의 기능 사일로 통과 시 증폭·왜곡
  - **Demand Sensing**: 시장 판매 신호를 zero-latency·no-bullwhip으로 부서 간 전달
  - **Harmonization**: 단위·SKU·위치 데이터를 부서 간 변환
  - **Orchestration**: 기능별 대안의 양방향 트레이드오프로 최적안 도출
  - **Planning Master Data**: 대체 소싱·사이클타임·환산율·원가·리드타임
  - **Synchronization**: 다른 사이클·호라이즌의 시스템·프로세스를 한 자리에 통합

### S&OP Pre-Pandemic State (p11-13)
- **효과성 하락**: 2019 35% effective vs 2016 65% — 30%p 급락
- **초기 구현 갭 Top 5** (그림 5):
  | 순위 | 갭 | % |
  |------|----|---|
  | 1 | 적시 데이터 확보 곤란 | **58%** |
  | 2 | 타 부서 지원 부족 | 54% |
  | 3 | 프로세스 지원 기술 부재 | 51% |
  | 4 | 임원 이해·지원 부족 | 43% |
  | 5 | 의사결정 명확성 부족 | 42% |

### Shifts in the Pandemic (p14)
- **Demand**: 주문 패턴 기반 수요계획은 노이즈만 증가. 소비(Consumption) 데이터 기반으로 전환, FVA(Forecast Value Added) 빈도 증가
- **Demand Sensing**: 슬러시 기간 보충 요건 모델링, 룰 기반을 zero-latency 센싱 로직으로 오버라이트
- **Digital Twin**: BlueCrux, Llamasoft, Logility, OMP — 시나리오 모델링·트레이드오프 시험
- **S&OP Execution Tech**: Aera Technology, Logility, Kinaxis — 유한 자원 정렬
- **공급사 협업**: 빈번한 데이터 공유, critical supplier playbook
- **재고 = 버퍼**: 회피 대상이 아닌 의도적 설계 대상으로 재정의

### What Does Good Look Like? (p15-17)
- **5가지 특성**: ①outside-in focus ②빠른 데이터 접근 ③프로세스 균형 ④조직 정렬 ⑤계획-실행 close-coupling
- **균형 현황** (p15): 36% OP 편중 + 21% S 편중 = **57% 불균형**, 41%만 균형
- **거버넌스 사례 (실패 케이스 3개)**:
  - $35B F&B: 33개 커스텀 S&OP 인스턴스, 예산 준수에 매몰 → 5년 후 어떤 지역도 "good"이 무엇인지 모름
  - $50B 화학사: 지역마다 호라이즌 정의 다름 → 글로벌 S&OP 롤업 불가
  - $22B 소비재사: 공급 신뢰성 결여로 부분 계획만 → 고객 서비스 갭
- **조직 정렬 갭 Top 4** (그림 7, Performance - Importance):
  | 순위 | 페어 | 중요 % | 실제 % | 갭 |
  |------|------|--------|--------|-----|
  | 1 | 신제품 개발 ↔ 운영·유통 | 78% | 39% | **−38%** |
  | 2 | 영업 ↔ 운영 | 96% | 60% | **−36%** |
  | 3 | 제조 ↔ 조달 | 94% | 70% | −24% |
  | 4 | 운영 ↔ IT | 74% | 50% | −23% |
- **계획-실행 연결**: 80%가 S&OP 있지만 **2명 중 1명만 plan→execution 연결** (12%만 tightly synced, 51%는 most-of-time 실행)

### Driving Process Maturity (p18) — 5단계 Maturity Model

| 차원 | Stage 1 | Stage 2 (Sales-Driven) | Stage 3 (IBP) | Stage 4 (Demand-Driven) | Stage 5 (Market-Driven) |
|------|---------|-------------------------|---------------|--------------------------|--------------------------|
| ALIGNMENT | 기능 중심 | "S" or "OP" 편중, 정렬 부재 | 부분 정렬, 전략 연결 약함 | 전략 기반 트레이드오프 합의 | 무역 파트너 통합, 시장-시장 |
| GOAL | 기능 KPI | 수요·공급 균형 | 비용 효율 최적 | 성장+서비스+재고 균형 | 성장+서비스+ROIC+재고 + 리스크 |
| CAPABILITY | S&OP 필요 인지만 | 수요관리 혼란 | 수요 흐름·제약 명확 | 다단위(달러/단위/등가) feasible plan | What-if·믹스 모델, 가시성, "Playbook" 정렬 |
| MEASUREMENT | 기능 메트릭만 | 조직 메트릭 등장 | 크로스펑셔널 메트릭 | 신뢰성·리스크·기회 균형 | 균형 스코어카드와 기능 매핑 |
| EXECUTION | 긴급 대응 | What-if 사일로 | What-if 등장하나 실행 미연결 | What-if 기반 Playbook + 실행 close-coupling | 클로즈드 루프 |
| REPORTING | Sales/Ops 기능 보고 | CFO·전략책임자 | BU CoE → 사업 리더 | BU CoE → 사업 리더 | Profit center manager |

→ **LAKA 진단**: 현재 Stage 1~2 사이 (S&OP 명시적 미구축, SCM 부문 10명 보유, 마케팅·글로벌사업 부서별 사일로). 18개월 목표 = Stage 3 진입.

### Why Is S&OP So Hard? (p19) — Capability Gap

| 능력 | 중요 % | 실제 % | 갭 |
|------|--------|--------|-----|
| What-if 분석으로 대안 도출 | 70% | 31% | **−39%** |
| 기회·리스크 관리 분석 | 85% | 49% | **−36%** |
| Sales-Operations 협업 | 88% | 55% | −33% |
| 최수익 계획 도출 기술 | 49% | 22% | −27% |
| 프로세스 흐름 관리 | 72% | 47% | −25% |

→ "갭 1·2위가 모두 분석 역량" — 정량 시뮬레이션 도구 없이 토론 프로세스에 머무는 게 본질적 문제.

### Selecting the Right Technology (p20-22)
- **5가지 happy user 특징**: ①Easy data access ②Fit (데이터 모델 적합도, 7%만 구매 전 테스트) ③Selection criteria (비즈니스 요건 vs IT 표준화) ④Culture (벤더 사용자 그룹·서비스) ⑤Evolution (벤더 리더십 일관성)
- **45%의 선택이 비즈니스 문제 미적합** — RFP보다 short-list + 실테스트 권장
- 90%+ 기업이 ERP 보유했지만 **18%만 필요한 데이터에 적시 접근**

### Recommendations (p23)
1. **컨설턴트가 만든 기술 회피** — 제품 관리·솔루션 개발 백그라운드 부재
2. **End-to-end 정의 명확화** — 공급사의 공급사부터 고객의 고객까지
3. **RFP 시간 낭비 회피** — 단축 short-list + 실테스트
4. **역량 빌딩에 집중** — 기술은 enabler, 목표는 capability
5. **Shiny object/유행 회피** — 성숙도 단계별 점진적 진화
6. **Real-time = 데이터 영역, 계획은 아님** — 계획을 real-time화하면 supply chain이 균형을 잃음

## Implement 핸드북과 비교

| 차원 | Implement (2014) | SCI (2020) |
|------|------------------|------------|
| 관점 | 컨설팅 베스트프랙티스 | 정량 서베이 + 벤더 평가 |
| 시간 호라이즌 | 4단계 (Strategic·S&OP·Tactical·Operational) | 4단계 (Strategic·Tactical·Operational·Executional) — 명칭 다름 |
| 월간 사이클 | 4주 사이클 (W1 Demand → W4 Execution) 명시 | 5-step (Business Building → Demand → Supply → Alignment → MT Review) |
| 성숙도 | Gartner 5단계 (React→Orchestrate) | 자체 5단계 (Sales-Driven→Market-Driven), Stage 3 = IBP |
| 투자 비중 | People 65% / Process 25% / System 10% | 변화관리 60% / 프로세스 30% / 기술 10% (사실상 동일) |
| 기술론 | "단순 프로세스 = Excel로 충분" | "ERP 90% 보유, 18%만 데이터 적시 접근. 45% 선정이 부적합" |
| 벤더 카탈로그 | 없음 | 33개 상세 평가 |
| Pothole/Pitfall | 없음 | 명시적 5+6 구분 |
| 주요 인용 가치 | 4주 사이클 설계, NPI 통합 (Ch5) | Maturity 진단, 조직 정렬 갭 정량값, 벤더 short-list |

→ **두 소스의 상호보완**: Implement = 프로세스 설계 (HOW), SCI = 진단·도구 선정 (WHERE/WHAT). W1·W6에서 두 소스 교차 인용, W7(LAKA v1) 작성 시 Implement 4주 사이클 + SCI 성숙도 진단 + 벤더 short-list 통합.
