## 0. 공통 운영 원칙 (5채널 전체 적용)

### 가. 4단 사이클을 모든 워크플로에 강제

[[품질팀_업무프로세스_2026-05]] 4종 워크플로 공통 패턴.

```
트리거 → 헤드 메시지 → 스레드 업데이트 → 클로즈
```

- **헤드 메시지**: 워크플로 시작 시점에 채널 본문으로 게시 (스레드 X).
  헤드 메시지가 없으면 그 워크플로는 채널 외부에서 진행 중인 것으로 간주.
- **스레드 업데이트**: 모든 진행 상황·이슈는 헤드 메시지 스레드 안에서만.
- **클로즈**: 워크플로 종료 시 헤드 메시지 스레드 마지막에 "[클로즈] 결과 + 공유폴더 링크" 1줄.

### 나. 본부장 옵저버 정책

- 본부장(천승범)은 5채널 모두 멤버.
- 채널에서 자동 가시화되므로 **개별 메일 포워딩 불요**.
- 본부장 멘션은 **에스컬레이션 룰에 명시된 사안만**. 일상 진행은 무멘션.

### 다. 외부 거래처 커뮤니케이션

- ODM·부자재 공급사·식약처 등 **외부와의 모든 공식 커뮤니케이션은 이메일/공문 유지**.
- Slack은 **외부 결과의 내부 트래킹·공유용**으로만.
- 외부에서 받은 공문·시험성적서는 **공유폴더 업로드 + 채널에 링크 게시**가 표준.

### 라. 정형 보고 헤드 메시지 — 채널별 템플릿

각 채널 섹션의 "📝 헤드 메시지 템플릿" 참조. 양식 강제 (자유 양식 헤드 금지).

### 마. DM / 외부 채널 / 전자결재 경계

- DM 지양 (1:1 케이스도 채널 스레드로).
- Slack ≠ 전자결재. 결재·정식 보관은 현 공유폴더·결재 시스템 그대로.
- 파일 첨부 시 제목에 날짜·버전 (`YYMMDD` 또는 `vN`).

```
🛎️ #qa-클레임-품질이슈 채널 가이드 (가안)

- Lot 번호 (필수)
- 제품명 / SKU
- 클레임 채널 (올영·자사몰·아마존·세포라·부츠·큐텐 등)
- 접수 일자

08 최종 업데이트

🚨 긴급 건 (식약처 위해정보 / 중대 클레임 / 회수 가능성)

   사후 채널 기록은 그 다음에 남기면 됩니다.
   천승범 010-8905-0366
   차승호 010-3552-3109
   김  건 010-5634-3274
```

### v1 → v2 변경 사유

- v1 "중대 클레임·회수·식약처 위해정보 = @천승범 + @이지철 즉시 멘션" 룰을 **전화·대면 우선**으로 변경
- **근거**: 슬랙 알림은 외근·미팅·집중 모드에서 놓칠 수 있음. 긴급 = 분 단위 대응 필요 (식약처 잠정중단 공문 1주 사이클). 전화·대면 = 즉시 인지 + 즉시 의사결정
- 슬랙 기록은 사후 트래킹용 (멘션 대신 일반 메시지 + 본부장 옵저버로 모니터링)

### 헤드 메시지 템플릿 (v1에서 유지)

```
[클레임 접수] 영업 담당 / 제품명 / 증상 요약 / 제조번호 / 접수일자

[자재 부적합] 자재명 / 자재번호 / 제조사 / 현장재고 / 라인 상태(중단/가동)

[위해정보] 출처(식약처/통관/외부) / 공문번호 / 대상 제품 / 1차 영향 파악
```

### 스레드 업데이트 / 외부 거래처 / 공유폴더 (v1 유지)

- 스레드: 클레임 8단계 각 단계 1줄, 게런티 "공문 발송 T+0 / 회신 대기 / 회수 T+N", 위해정보 시간순
- 외부 거래처 (한국콜마·코코스팩·기타 부자재): 이메일·공문 유지. 공문 회수 즉시 공유폴더 업로드 + 본 채널 스레드 링크
- 공유폴더: 클레임 접수증·처리결과서·게런티 공문은 공유폴더 보관 필수 (분쟁 대비 서면 근거)

Owner: 차승호 / 본부장 옵저버
