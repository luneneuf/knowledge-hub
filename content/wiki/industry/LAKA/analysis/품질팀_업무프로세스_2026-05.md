# 품질팀 업무 프로세스 도식

> **목적**: 천승범 본부장이 5/20 받은 4건의 진행 중 케이스를 기반으로, 품질팀이 실제로 수행하는 4종 업무 워크플로를 시퀀스 도식으로 정리.
> **사용처**: [[품질팀_슬랙채널_규칙_2026-05]] 의 각 채널 운영 룰의 근거.
> **근거 케이스(2026-05-20 본부장 일괄 포워딩 4건)**:
> - 씨앤씨인터내셔널 Audit 개선요청 (차승호, 2026-04-27 ~ 2026-05-29)
> - 매직립픽서 상세페이지 광고문안 워싱본 검토 (김건 ↔ BM팀 최민녕, 2026-05-14 ~ 2026-05-19)
> - 매직립픽서 단상자 표시문안 검수 (김건 ↔ 디자인팀 박다빈, 2026-04-24)
> - LAK-PGTT 용기 부자재 게런티 공문 (차승호·한국콜마·코코스팩, 2026-05-07 ~ 2026-05-18)

## 2. 워크플로 A — ODM 제조사 Audit

### 단계 요약

| 단계 | 일정(상대) | 담당 | 산출물 |
|------|-----------|------|--------|
| 1. 사전 자체평가 요청 | T-2주 | 차승호 | 체크시트, 정제수 도면·SOP LIST 요청 공문 |
| 2. 현장 점검 (5대 영역) | D-day (반일) | 차승호·김건 (품질) + 조민경 (구매) | 현장 점검 노트 |
| 3. 등급 판정 + 개선지시서 작성 | T+2주 | 차승호 | 평가등급(A~D), 개선지시서 (.pptx + .doc) |
| 4. 개선요청 공문 발송 | T+2주 | 차승호 | 개선요청 공문 (회신기한 4주 명시) |
| 5. 개선결과서 회신 회수 | T+6주 | 차승호 | 제조사 개선결과서 |
| 6. 클로즈 | T+6~8주 | 차승호 | 공유폴더 보관, 다음 사이클 등록 |

### 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant QA as LAKA 품질팀<br/>(차승호·김건)
    participant PUR as LAKA 구매팀<br/>(조민경)
    participant ODM as ODM 제조사<br/>(예: 씨앤씨인터내셔널)
    participant SLK as #qa-제조업체-audit
    participant HEAD as 천승범 본부장<br/>(옵저버)

    Note over QA,ODM: T-2주 — 사전 자체평가
    QA->>ODM: 체크시트 + 요청서류 (정제수 도면·SOP LIST) [이메일]
    ODM->>QA: 자체평가서 회신 [이메일]
    QA->>SLK: 헤드 메시지: [Audit 예정] 업체·일자·실시자

    Note over QA,ODM: D-day — 현장 점검
    PUR-->>ODM: 현장 동행 (구매팀 참여)
    QA->>ODM: 5대 영역 점검<br/>(공장 / 생산·품질·보관 / 시험·문서 / 협력업체 / 외주)
    QA->>SLK: 스레드 업데이트: 진행 상황·특이사항

    Note over QA,ODM: T+2주 — 개선요청
    QA->>QA: 등급 판정 + 개선지시서 작성
    QA->>ODM: [공문] 등급·판정·개선지시서 + 회신기한 4주
    QA->>SLK: 스레드: 등급·마감일 핀

    alt 부적합 등급 (D 이하)
        QA->>HEAD: @멘션 (에스컬레이션)
    end

    Note over QA,ODM: T+6주 — 회신 회수
    ODM->>QA: 개선결과서 [이메일]
    QA->>SLK: 클로즈: 결과 요약 + 공유폴더 링크
```

## 3. 워크플로 B — 광고문안 워싱본 검토

### 단계 요약

| 단계 | 담당 | 산출물 |
|------|------|--------|

| 5. 최종 검토 OK | 김건 | 검토완료 통보 |

### 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant BM as 프로덕트 부문<br/>BM팀
    participant QA as 품질팀<br/>(김건)
    participant SLK as #qa-신제품-출시검증
    participant HEAD as 본부장 (옵저버)

    BM->>QA: 워싱본 PPT 검토 요청 [이메일 + d6200 그룹]
    BM->>SLK: 헤드 메시지: [광고문안] 제품·요청자·회신희망일

    loop 합의될 때까지 (보통 1~2 라운드)
        QA->>QA: 식약처 표시광고 가이드 대조 + 수정요청 정리
        QA->>BM: 수정요청 회신 [이메일]
        QA->>SLK: 스레드 업데이트: 수정요청 N건 회신
        BM->>QA: 수정본 + 추가 확인사항 [이메일]
    end

    QA->>BM: 최종 검토 OK [이메일]
    QA->>SLK: 클로즈: [광고문안 검토완료] 제품·일자
    Note right of SLK: 본부장은 채널에서 자동 가시화<br/>별도 포워딩 불요
```

## 4. 워크플로 C — 표시문안 (단상자) 검수

### 단계 요약

| 단계 | 담당 | 산출물 |
|------|------|--------|
| 1. 단상자 문안 검수 요청 | 디자인팀 (요청자) | 한:영문 병기 PDF |
| 2. 화장품법 표시기재사항 대조 | 김건 | 수정요청 항목 |
| 3. 수정요청 회신 | 김건 | 정정 항목 통보 (제조업자·중량·바코드 등) |
| 4. 수정본 재검수 | 디자인팀 → 김건 | 수정 PDF |
| 5. 검수 완료 OK = 인쇄 발주 GO | 김건 | 검수완료 통보 |

### 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant DS as 프로덕트 부문<br/>디자인팀
    participant QA as 품질팀<br/>(김건)
    participant BM as BM팀<br/>(참조)
    participant SLK as #qa-신제품-출시검증

    DS->>QA: 단상자 한:영문 병기 PDF 검수 요청<br/>(BM팀 참조) [이메일]
    DS->>SLK: 헤드 메시지: [표시문안] 제품·단상자 사양·회신희망일

    QA->>QA: 화장품법 표시기재사항 대조<br/>(제조업자·중량·바코드·성분·주의사항)
    QA->>DS: 수정요청 회신 (BM·SCM 품질팀 참조)
    QA->>SLK: 스레드: 수정 N건 — 항목 리스트

    DS->>QA: 수정본 재검수 요청
    QA->>DS: 검수 완료 OK
    QA->>SLK: 클로즈 + 인쇄 발주 GO 시그널
```

| 단계 | 실제 일자 | 수정 항목 |
|------|---------|-----------|
| 1. 검수 요청 (디자인 박다빈) | 2026-04-24 14:02 | PDF 52.62MB (대용량 첨부) |
| 2. 수정요청 회신 (김건) | 2026-04-24 17:00 | 3건 — 제조업자(한국콜마 부천공장) / 중량(0.03→0.13 oz) / 바코드 추가(8809611869608) |
| 3. 본부장 포워딩 | 2026-05-20 09:57 | — |

**용어 메모**: "검수"(B 검수) ≠ "검토"(B 검토).
- 검수 = 단상자·라벨의 **법령 표시기재사항** (제조업자·성분·중량 등) 정확성 점검
- 검토 = 광고 문구의 **표시광고 가이드 부합성** 점검

→ 같은 제품(매직립픽서)에도 두 워크플로가 모두 흐름. 두 건은 채널은 동일(`#qa-신제품-출시검증`)하지만 스레드 별개로 운영.

## 5. 워크플로 D — 부자재 게런티 / 부적합

### 단계 요약

| 단계 | 담당 | 산출물 |
|------|------|--------|
| 1. 불량 발견 → 라인 격리 | 한국콜마 (위탁제조) | 불량 보고서, 샘플 사진 |
| 2. LAKA 통보 | 한국콜마 → 차승호 | 불량 보고 이메일 |
| 3. 게런티 요청 공문 발송 | 한국콜마 → 부자재 공급사 | 공문 (자재번호·불량양상·책임범위·서면회신 요청) |
| 4. 재고 수량 확정 통보 | 한국콜마 → 부자재 공급사 | 보유·현장 수량 |
| 5. 게런티 공문 회신 | 부자재 공급사 → LAKA + 한국콜마 | 품질보증 공문 (직인 포함) |
| 6. 시장 클레임 발생 시 | (조건부) | 공문 근거로 책임 추궁 |

### 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant KOL as 한국콜마<br/>(위탁 제조)
    participant SUP as 부자재 공급사<br/>(예: 코코스팩)
    participant QA as LAKA 품질팀<br/>(차승호)
    participant SLK as #qa-클레임-품질이슈
    participant HEAD as 본부장 (옵저버)

    Note over KOL: 라인 가동 중 불량 발견
    KOL->>KOL: 불량 자재 격리·라인 중단<br/>샘플 사진 확보
    KOL->>QA: 불량 보고 + 게런티 요청 의사 공유 [이메일]
    QA->>SLK: 헤드: [자재 부적합] 자재명·번호·제조사·현장재고<br/>@천승범 (라인 중단 사안)

    KOL->>SUP: [공문] 게런티 요청<br/>(자재번호·불량양상·책임범위 명시)
    KOL->>QA: 공문 사본 + 재고 수량 통보
    QA->>SLK: 스레드: 공문 발송 / 회신 대기 (T+N일)

    SUP->>KOL: [공문] 품질보증 회신 (직인 포함)
    SUP->>QA: 공문 사본 송부
    QA->>SLK: 스레드: 게런티 공문 회수 — 책임범위 N만개
    QA->>HEAD: @천승범 (분쟁 대비 서면 확보 시점 알림)

    alt 시장 클레임 발생 시
        Note over QA,SUP: 공문 근거로 책임 추궁
    end
```

## 7. 4종 워크플로 공통 패턴

1. **트리거 → 헤드 메시지 → 스레드 업데이트 → 클로즈** 4단 사이클
