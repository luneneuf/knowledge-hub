## 1. 목표 및 비목표

### 목표 (1단계 종료 시점에 달성)

1. Slack 워크스페이스에 신규 채널 `#qa-시그널-피드` 개설
2. RSS·이메일 기반으로 **국내·외 안전관리 시그널 6~8개 소스를 채널에 자동 게시**
3. 김건 책임이 매일 채널을 5~10분 검수 → 중요 항목을 [[품질팀_슬랙운영_초안_2026-05|`#qa-인허가-법규`]]·`#qa-클레임-품질이슈`로 이관
4. 4주 운영 후 **본부장 회고 → 2단계(AI 필터·요약) 진입 여부 결정**

### 비목표 (1단계에서는 다루지 않음)

- AI 자동 요약·관련성 분류 (2단계)
- LAKA 브랜드명·제품명 자동 키워드 매칭 알림 (2단계)
- SNS·리뷰 사이트 시그널 수집 (3단계)
- 별지 1호 서식 자동 채우기 (3단계)
- 식약처 안전성 정기보고 자동 제출 (영구 비목표)

### 3.3 RSS 소스 명세 (1단계 구독 대상 8개)

> ⚠️ 각 URL은 셋업 시점에 실제 동작 검증 필요. 일부 사이트는 RSS 미제공 → 대체 수단 명시.

#### A. RSS 직접 제공 — Slack `/feed subscribe` 명령으로 즉시 등록

| # | 출처 | RSS URL (예상/확인 필요) | 예상 게시 빈도 | 우선순위 |
|---|------|-------------------------|---------------|----------|
| 1 | **FDA Recalls — Cosmetics** | `https://www.fda.gov/about-fda/contact-fda/stay-informed/rss-feeds/recalls/rss.xml` (전체) → Slack에서 카테고리 필터 또는 검수 단계 필터 | 주 1~5건 | 🔴 높음 |
| 2 | **EU Safety Gate Alerts** | `https://ec.europa.eu/safety-gate-alerts/screen/...` (RSS 링크 페이지에서 화장품 카테고리 선택) | 주 5~20건 (전체 카테고리, 화장품 비중 약 5%) | 🟡 중간 |
| 3 | **PubMed — cosmetic adverse events** | `https://pubmed.ncbi.nlm.nih.gov/rss/search/...` (검색식: `(cosmetic[Title]) AND (adverse OR allergy OR dermatitis OR recall)`, RSS 자동 생성) | 주 5~15건 | 🟢 낮음 |
| 4 | **식약처 보도자료** | `https://www.mfds.go.kr/brd/m_99/...rss` (보도자료 RSS — 회수 공고는 별도 페이지에 있으나 보도자료에 회수·판매중지 발표가 동반되는 경우 多) | 주 5~10건 | 🔴 높음 |

#### B. RSS 미확인 또는 미제공 — 셋업 시 대체 수단 확인 필요

| # | 출처 | 1차 시도 | 미제공 시 대체 |
|---|------|----------|----------------|
| 5 | **식약처 의약품안전나라 — 화장품 회수·판매중지 공고** | RSS 페이지 검색 (nedrug.mfds.go.kr) | **IFTTT** "Website RSS" 또는 "If page changes" 트리거 → Slack 채널 이메일 주소로 전송. 또는 매일 김건 책임이 사이트 직접 방문 (수동) |
| 6 | **대한화장품협회 (KCIA) 공지사항** | https://www.kcia.or.kr 공지사항 RSS 페이지 검색 | IFTTT or 수동 |
| 7 | **UK OPSS Product Safety Database** | https://www.gov.uk/government/organisations/office-for-product-safety-and-standards 게시판 RSS | IFTTT or 수동 |
| 8 | **일본 PMDA 화장품 회수** | https://www.pmda.go.jp 회수정보 페이지 RSS | IFTTT or 수동 (한국어 번역 부담 있어 우선순위 가장 낮음) |

#### C. 1단계에서 제외 (2단계로 이월)

- Google Alerts (브랜드명·키워드) — 이메일 → Slack 자동 전송은 Gmail 필터 + 별도 도구 필요. 1단계에서는 김건 책임이 Gmail 알림 직접 확인
- 화해·@cosme·세포라·부츠 리뷰 — 스크래퍼 필요 (2단계)
- TikTok·X·인스타그램 — API 또는 SaaS 필요 (3단계)

### 3.4 Slack RSS 앱 활용 — 구체 셋업 단계

> Slack의 기본 내장 RSS 통합 사용 (별도 결제 불필요)

1. Slack 워크스페이스 관리자가 RSS 앱 활성화
   - Slack > "Add apps" > "RSS" 검색 > "Add to Slack"
2. `#qa-시그널-피드` 채널 입장 후 다음 명령 실행 (RSS 직접 제공 4개부터):
   ```
   /feed subscribe https://www.fda.gov/about-fda/contact-fda/stay-informed/rss-feeds/recalls/rss.xml
   /feed subscribe [EU Safety Gate Cosmetics RSS URL]
   /feed subscribe [PubMed search RSS URL]
   /feed subscribe [식약처 보도자료 RSS URL]
   ```
3. RSS 미제공 4개 출처는 다음 중 한 가지로 처리 (셋업 시 결정):
   - **옵션 1 — IFTTT 무료 플랜**: "If website RSS feed updates" 또는 "If page changes" 트리거 → "Post to Slack channel" 액션
   - **옵션 2 — 김건 책임 수동**: 평일 매일 오전 9시 5개 사이트 1회 방문 (10분), 새 항목은 김건 책임이 채널에 직접 게시
   - **옵션 3 — Gmail 필터 + 채널 이메일**: 해당 사이트가 이메일 알림 제공 시 Gmail 라벨 + 자동 전달
4. 구독 확인: `/feed list` 명령으로 모든 구독 출처 확인

### 3.5 메시지 포맷 표준

#### 자동 게시 메시지 (Slack RSS 봇이 자동 생성)

자동 게시물은 봇이 생성한 그대로 둔다. 김건 책임이 검수 후 다음 라벨 이모지 추가:

| 이모지 | 의미 | 추가 액션 |
|--------|------|----------|
| 🔴 | Tier A — LAKA 직접 영향 가능성 | 스레드에 1~2문장 요약 작성 + 차승호·천승범·필요시 이지철 멘션 + 이관 채널로 링크 공유 |
| 🟡 | Tier B — 모니터링 (카테고리·경쟁사·ODM) | 스레드에 1문장 코멘트 |
| 🟢 | Tier C — 일반 동향 (학술·정책 변경 등) | 별도 액션 없음. 주간 회고 시 정리 |
| ✅ | 검수 완료 (위 셋 중 어느 하나도 해당 안 됨) | 액션 없음. 검수 흔적만 |
| 🇰🇷 / 🇺🇸 / 🇬🇧 / 🇯🇵 / 🇪🇺 | 관할국 표시 | 어느 시장 관련인지 빠른 식별 |

#### Tier A 이관 시 메시지 템플릿 (`#qa-인허가-법규` 또는 `#qa-클레임-품질이슈`)

```
🔴 [시그널 이관] 출처: <원본 링크>

요약: <1~2문장 한국어 요약>

LAKA 영향 가능성:
- 제품: <영향 가능 SKU 또는 카테고리>
- 사유: <ODM 공통·향료·색소·법규 등>

제안 액션:
- <누가 무엇을 언제까지>

원본 채널: #qa-시그널-피드 <원본 메시지 영구 링크>

@천승범 @차승호 (@이지철 — 회수 가능성·세포라 영향 시)
```

  2. 🔴 / 🟡 / 🟢 / ✅ 분포
  3. 🔴 → 이관 → 실제 조치된 건수
  4. RSS 소스별 게시 건수 (편향 여부 확인)
  5. 검수 시간 평균 (목표: 10분/일 이내)
  6. **누락 사례** — 외부에서 이미 발생했으나 채널에 잡히지 않았던 시그널 (예: 다른 채널·이메일·SNS에서 먼저 알게 된 안전 이슈)
  7. RSS 소스 추가·제거 제안

### 10.2 IFTTT 셋업 가이드 (RSS 미제공 사이트용)

1. https://ifttt.com 회원가입 (무료 플랜)
2. Slack 워크스페이스 연결 (Settings > Connections > Slack)
3. 새 Applet 생성:
   - **If This** = "Webhooks" 또는 "RSS Feed" (사이트가 RSS 부분 제공 시) 또는 "If Web Page Changes" (Pro 필요)
   - **Then That** = "Slack — Post to channel" — 채널: `#qa-시그널-피드`, 메시지: `New update from {{출처}}: {{Title}} — {{Link}}`
4. 활성화 후 3일간 트리거 확인
