# UWO / 역사 Wiki Schema

## 이 위키의 목적

대항해시대 오리진(UWO) 게임 기반 역사 지식 베이스. 역사 인물, 사건, 게임 데이터를 엔티티·개념 단위로 축적한다. 리서치한 지식을 트위터·블로그·마이크로사이트 콘텐츠로 전환하는 파이프라인의 기반.

소스 1개 = 요약 1개가 아니라, 소스에서 추출한 지식이 해당 엔티티·개념 페이지로 분산 통합된다.

## 폴더 구조

raw/          — 원본 소스. 절대 수정 금지. 불변.
wiki/
  entities/   — 역사 인물, 장소, 조직, 게임 항해사 (고유명사)
  concepts/   — 항해술, 교역 네트워크, 역사 이론 (추상 개념)
  events/     — 전투, 원정, 조약, 역사적 사건
  comparisons/ — 비교 분석 (동서양 항해술 비교 등)
  guides/     — content_style.md 등 운영 지침
  content/    — 콘텐츠 초안·발행 기록 (트위터, 블로그, 마이크로사이트)
index.md      — 전체 카탈로그
log.md        — 작업 로그 (시간순, append-only)

## 인제스트 워크플로

소스 1개를 처리할 때 아래 6단계를 순서대로 수행한다.

### 1단계 — 소스 읽기 & 지식 목록 추출
소스에서 다음을 뽑는다:
- 엔티티 (인물, 장소, 조직, 선박, 게임 항해사)
- 개념 (항해술, 교역 구조, 역사 이론)
- 사건 (전투, 원정, 조약)
- 관계 ("A는 B의 부관이다", "A는 B 전투에 참전했다")
- 수치/데이터 (연도, 병력, 교역량 등)

추출 목록을 먼저 보여주고 확인받는다.

### 2단계 — 기존 페이지 매핑
index.md를 읽고 각 항목이 이미 위키에 있는지 확인.
- 있으면 → 해당 페이지 업데이트
- 없으면 → 새 페이지 생성

### 3단계 — 페이지 업데이트 / 생성
업데이트 시:
- 기존 내용과 충돌하면 `> ⚠️ 출처 간 충돌:` 블록으로 양쪽 모두 남김
- 새 정보는 해당 섹션 뒤에 추가, `[^소스명]` 각주 표기
- frontmatter sources 리스트에 소스명 추가
- updated 날짜 갱신

절대 하지 않는 것:
- src_소스명.md 같은 소스별 요약 페이지 생성 금지
- 소스의 모든 내용은 엔티티/개념/사건 페이지로 분산 통합

### 4단계 — 위키링크 [[]] 삽입
다른 위키 페이지로 연결 가능한 고유명사에 [[링크]] 삽입.
index.md에 있는 페이지만 링크한다 — 없는 페이지를 링크하지 않는다.

### 5단계 — index.md 업데이트
새 페이지는 해당 섹션에 추가. 총 페이지 수 갱신. 날짜 갱신.

### 6단계 — log.md 기록
## [YYYY-MM-DD] ingest | 소스명
- 생성/업데이트한 페이지 목록
- 발견된 충돌 사항

## 콘텐츠 스타일 참조

UWO 콘텐츠 작성 시 반드시 [[content_style]]을 참조한다.
핵심: 트위터 스레드 7단계 훅 구조 (훅 → 배경 → 에피소드 → 반전 → 게임 연결)
톤: 구어체, 아이러니 훅, 역사 우선·게임 부록
금지: 역사 왜곡, 감탄사 남발, 게임 광고성 오프닝

## 검색 규칙 (QMD 도입 전)

1. 위키 검색 시 index.md를 먼저 읽는다.
2. 관련 페이지만 열어서 컨텍스트로 사용한다.
3. wiki/ 전체를 한 번에 읽지 않는다.

## Linear 연동

- 팀: UWO (prefix: UWO-)
- 위키 인제스트 이슈: [위키] 제목
- 콘텐츠 작업 이슈: [콘텐츠] 제목
- 작업 완료 시 "UWO-N 완료" 보고

## 금지 사항

1. src_*.md 소스별 요약 페이지 생성 금지
2. raw/ 파일 수정 금지
3. 존재하지 않는 페이지로 [[링크]] 걸기 금지
4. 인제스트 없이 index.md 수동 편집 금지
5. 역사적 사실을 검증 없이 창작하지 않는다
6. 게임 데이터와 역사적 사실을 혼동하지 않는다 — 항상 구분하여 서술
